# 多环境配置文件说明

## 📋 配置文件列表

```
src/main/resources/
├── application.yml          # 公共配置（所有环境共享）
├── application-dev.yml      # 开发环境配置
├── application-test.yml     # 测试环境配置
└── application-prod.yml     # 生产环境配置
```

---

## 🎯 环境切换方式

### 方法 1：修改主配置文件（推荐用于本地开发）

编辑 `application.yml`，修改激活的环境：

```yaml
spring:
  profiles:
    active: dev  # 改为 test 或 prod
```

---

### 方法 2：命令行参数（推荐用于部署）

启动时通过命令行参数指定环境：

```bash
# 开发环境
java -jar vllmpt.jar --spring.profiles.active=dev

# 测试环境
java -jar vllmpt.jar --spring.profiles.active=test

# 生产环境
java -jar vllmpt.jar --spring.profiles.active=prod
```

---

### 方法 3：环境变量

设置环境变量：

```bash
# Linux/Mac
export SPRING_PROFILES_ACTIVE=prod
java -jar vllmpt.jar

# Windows PowerShell
$env:SPRING_PROFILES_ACTIVE="prod"
java -jar vllmpt.jar
```

---

### 方法 4：IDEA 运行配置

在 IntelliJ IDEA 中：

1. 打开 Run → Edit Configurations
2. 选择你的应用配置
3. 在 **Override Spring Boot profile** 中填写环境名
4. 或在 **Environment variables** 中添加 `SPRING_PROFILES_ACTIVE=prod`

## 📊 各环境配置差异

### 1. 开发环境 (dev)

**适用场景**：本地开发、调试

| 配置项 | 值 | 说明 |
|--------|------|------|
| MySQL | `vllmpt_dev` | 开发数据库 |
| Redis | localhost:6379 | 本地 Redis |
| RabbitMQ | guest/guest | 默认账号 |
| MinIO | localhost:9000 | 本地存储 |
| ES | localhost:9200 | 本地搜索引擎 |
| API Key | 测试 Key | 硅基流动测试密钥 |
| 日志级别 | TRACE/DEBUG | 最详细日志 |
| Swagger | ✅ 开启 | 方便 API 测试 |
| 监控端点 | ✅ 全部开启 | 便于调试 |

---

### 2. 测试环境 (test)

**适用场景**：功能测试、集成测试

| 配置项 | 值 | 说明 |
|--------|------|------|
| MySQL | `vllmpt_test` | 测试数据库 |
| Redis | 独立测试服务器 | 与开发隔离 |
| RabbitMQ | 独立测试服务器 | 消息队列 |
| MinIO | 独立测试服务 | 文件存储 |
| ES | 独立测试实例 | 日志检索 |
| API Key | 测试专用 | 测试环境密钥 |
| 日志级别 | INFO/DEBUG | 适中日志 |
| Swagger | ❌ 关闭 | 不需要 API 文档 |
| 监控端点 | 部分开启 | health, info, metrics |

---

### 3. 生产环境 (prod)

**适用场景**：正式环境、用户访问

| 配置项 | 值 | 说明 |
|--------|------|------|
| MySQL | 通过环境变量配置 | 高可用集群 |
| Redis | 通过环境变量配置 | 支持集群模式 |
| RabbitMQ | 通过环境变量配置 | 集群模式 |
| MinIO | 通过环境变量配置 | S3 兼容存储 |
| ES | 通过环境变量配置 | 集群模式 |
| API Key | 环境变量 `${SILICONFLOW_API_KEY}` | 严格保密 |
| 日志级别 | WARN/INFO | 最小化日志 |
| Swagger | ❌ 关闭 | 安全问题 |
| 监控端点 | 严格管控 | 仅 health, info |
| SSL | ✅ 启用 | HTTPS 加密 |
| 错误页面 | 自定义 | 不暴露系统信息 |

## 🔐 敏感信息管理

### 生产环境最佳实践

#### 1. 使用环境变量

```yaml
spring:
  datasource:
    username: ${DB_USERNAME}      # 从环境变量读取
    password: ${DB_PASSWORD}      # 从环境变量读取
```

#### 2. 使用配置文件模板

创建 `.env.example` 文件（不包含真实密码）：

```bash
# 数据库
DB_HOST=localhost
DB_PORT=3306
DB_NAME=vllmpt_prod
DB_USERNAME=root
DB_PASSWORD=CHANGE_ME

# Redis
REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_PASSWORD=CHANGE_ME

# MySQL 密码示例
RABBITMQ_USERNAME=guest
RABBITMQ_PASSWORD=CHANGE_ME

# MinIO
MINIO_ENDPOINT=http://localhost:9000
MINIO_ACCESS_KEY=minioadmin
MINIO_SECRET_KEY=CHANGE_ME
MINIO_BUCKET=vllmpt-prod-images

# Elasticsearch
ELASTICSEARCH_HOSTS=localhost:9200
ELASTICSEARCH_USERNAME=elastic
ELASTICSEARCH_PASSWORD=CHANGE_ME

# AI API
SILICONFLOW_API_KEY=sk-CHANGE_ME

# 服务器
SERVER_PORT=8080
```

#### 3. 实际使用时

复制并修改：

```bash
cp .env.example .env
# 编辑 .env 文件，填入真实值

# 加载环境变量
export $(cat .env | grep -v '^#' | xargs)
java -jar vllmpt.jar --spring.profiles.active=prod
```

## 🚀 快速启动命令

### 开发环境

```bash
# IDEA 直接运行（默认使用 dev 环境）

# 或命令行启动
mvn spring-boot:run -Dspring-boot.run.profiles=dev
```

---

### 测试环境

```bash
# 本地测试
mvn spring-boot:run -Dspring-boot.run.profiles=test

# JAR 包启动
java -jar target/vllmpt.jar --spring.profiles.active=test
```

---

### 生产环境（Docker）

```bash
docker run -d \
  -e SPRING_PROFILES_ACTIVE=prod \
  -e DB_HOST=mysql-server \
  -e DB_USERNAME=vllmpt_user \
  -e DB_PASSWORD=your_password \
  -e REDIS_HOST=redis-server \
  -e REDIS_PASSWORD=your_redis_password \
  -e RABBITMQ_HOST=rabbitmq-server \
  -e RABBITMQ_USERNAME=vllmpt \
  -e RABBITMQ_PASSWORD=your_rabbitmq_password \
  -e MINIO_ENDPOINT=http://minio:9000 \
  -e MINIO_ACCESS_KEY=minioadmin \
  -e MINIO_SECRET_KEY=your_minio_password \
  -e ELASTICSEARCH_HOSTS=elasticsearch:9200 \
  -e ELASTICSEARCH_USERNAME=elastic \
  -e ELASTICSEARCH_PASSWORD=your_es_password \
  -e SILICONFLOW_API_KEY=your_siliconflow_api_key \
  -p 8080:8080 \
  vllmpt:latest
```

---

### 生产环境（Systemd 服务）

创建 `/etc/systemd/system/vllmpt.service`：

```ini
[Unit]
Description=Vllmpt Application
After=syslog.target network.target

[Service]
Type=simple
User=vllmpt
Group=vllmpt
WorkingDirectory=/opt/vllmpt
ExecStart=/usr/bin/java -jar /opt/vllmpt/vllmpt.jar --spring.profiles.active=prod
SuccessExitStatus=143
Restart=always
RestartSec=10
StandardOutput=journal
StandardError=journal

LimitNOFILE=65536
LimitNPROC=4096

Environment=JAVA_HOME=/usr/lib/jvm/java-21-openjdk
Environment=DB_HOST=db-server
Environment=DB_USERNAME=vllmpt_user
Environment=DB_PASSWORD=your_password
Environment=REDIS_HOST=redis-server
Environment=REDIS_PASSWORD=your_redis_password
Environment=RABBITMQ_HOST=rabbitmq-server
Environment=RABBITMQ_USERNAME=vllmpt
Environment=RABBITMQ_PASSWORD=your_rabbitmq_password
Environment=MINIO_ENDPOINT=http://minio:9000
Environment=MINIO_ACCESS_KEY=minioadmin
Environment=MINIO_SECRET_KEY=your_minio_password
Environment=ELASTICSEARCH_HOSTS=elasticsearch:9200
Environment=ELASTICSEARCH_USERNAME=elastic
Environment=ELASTICSEARCH_PASSWORD=your_es_password
Environment=SILICONFLOW_API_KEY=your_siliconflow_api_key

[Install]
WantedBy=multi-user.target
```

启动服务：

```bash
sudo systemctl daemon-reload
sudo systemctl start vllmpt
sudo systemctl enable vllmpt  # 开机自启
sudo journalctl -u vllmpt -f   # 查看日志
```

## 🔧 常见问题

### Q1: 如何查看所有可用的配置属性？

```bash
./mvnw spring-boot:run -Dspring-boot.run.arguments=--debug
```

---

### Q2: 如何排查配置问题？

```bash
# 显示激活的 profile
java -jar vllmpt.jar --spring.profiles.active=test --debug 2>&1 | grep "Active profiles"
```

---

### Q3: 如何临时覆盖某个配置？

```bash
# 运行时单独覆盖端口
java -jar vllmpt.jar --server.port=9090 --spring.profiles.active=test
```

---

### Q4: 配置文件优先级是怎样的？

```
1. 命令行参数 (--key=value)          ← 最高
2. Java 系统属性 (-Dkey=value)
3. 操作系统环境变量
4. application-{profile}.yml
5. application.yml                   ← 最低
```
