# 会话服务（Chat Service）开发计划

## 📋 服务概述

### 服务职责
会话服务是电商智能助手平台的核心服务，负责管理用户与 AI 的多轮对话，包括会话创建、消息收发、上下文管理、历史记录查询等功能。

### 核心功能
- ✅ 会话生命周期管理（创建、查询、删除）
- ✅ 多轮对话上下文保持
- ✅ 消息发送和接收（支持文本、图片）
- ✅ 历史会话检索（MySQL + Elasticsearch）
- ✅ 会话状态同步（Redis）
- ✅ 流式响应支持（SSE）

### 技术栈
- **框架**：Spring Boot 3.x + Java 21
- **数据库**：MySQL 8.0 + MyBatis
- **缓存**：Redis 7.x
- **搜索**：Elasticsearch 8.x
- **消息队列**：RabbitMQ 3.x
- **HTTP 客户端**：WebClient（响应式）

#### 2. 核心表结构（简化版）

```sql
CREATE TABLE conversation (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    session_id VARCHAR(64) NOT NULL UNIQUE,  -- 会话 ID（UUID）
    user_id BIGINT NOT NULL,                  -- 用户 ID
    title VARCHAR(200),                       -- 会话标题（自动生成）
    model_type VARCHAR(20) DEFAULT 'text',    -- 模型类型
    message_count INT DEFAULT 0,              -- 消息数量
    status TINYINT DEFAULT 1,                 -- 1:活跃 0:已归档
    last_message_at DATETIME,                 -- 最后消息时间
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_user_id (user_id),
    INDEX idx_session_id (session_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

**消息表（message）**
```sql
CREATE TABLE message (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    session_id VARCHAR(64) NOT NULL,          -- 会话 ID
    message_id VARCHAR(64) NOT NULL UNIQUE,   -- 消息 ID（UUID）
    role VARCHAR(10) NOT NULL,                -- user/assistant/system
    content_type VARCHAR(20) DEFAULT 'text',  -- text/image/mixed
    content TEXT,                             -- 文本内容
    image_urls JSON,                          -- 图片 URL 数组
    tokens_used INT,                          -- Token 消耗
    metadata JSON,                            -- 元数据（模型名称、温度等）
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_session_id (session_id),
    INDEX idx_created_at (created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 🔧 技术要点

#### 1. Redis Key 设计规范

```
# 会话基本信息
Key: session:info:{sessionId}
Value: JSON(SessionDTO)
TTL: 1800秒（30分钟）

# 会话消息列表
Key: session:messages:{sessionId}
Value: JSON(List<Message>)
TTL: 600秒（10分钟）

# 用户会话列表（最近 50 个）
Key: user:sessions:{userId}
Value: ZSet<sessionId, timestamp>
TTL: 永久（定期清理）

# 会话锁（防止并发修改）
Key: lock:session:{sessionId}
Value: UUID
TTL: 10秒
```

#### 2. Cache-Aside 模式实现

```java
@Service
public class SessionCacheService {
    
    @Autowired
    private RedisTemplate<String, Object> redisTemplate;
    
    @Autowired
    private SessionService sessionService;
    
    /**
     * 获取会话详情（带缓存）
     */
    public SessionVO getSessionWithCache(String sessionId) {
        String key = "session:info:" + sessionId;
        
        // 1. 先查缓存
        SessionVO cached = (SessionVO) redisTemplate.opsForValue().get(key);
        if (cached != null) {
            log.debug("缓存命中: {}", sessionId);
            return cached;
        }
        
        // 2. 缓存未命中，查数据库
        log.debug("缓存未命中，查询数据库: {}", sessionId);
        SessionVO session = sessionService.getSessionDetail(sessionId);
        
        // 3. 写入缓存
        if (session != null) {
            redisTemplate.opsForValue().set(key, session, 30, TimeUnit.MINUTES);
        }
        
        return session;
    }
    
    /**
     * 更新会话缓存
     */
    public void updateSessionCache(String sessionId, SessionVO session) {
        String key = "session:info:" + sessionId;
        redisTemplate.opsForValue().set(key, session, 30, TimeUnit.MINUTES);
    }
    
    /**
     * 删除会话缓存
     */
    public void deleteSessionCache(String sessionId) {
        String infoKey = "session:info:" + sessionId;
        String messagesKey = "session:messages:" + sessionId;
        redisTemplate.delete(Arrays.asList(infoKey, messagesKey));
    }
}
```

#### 3. 消息列表缓存更新策略

**问题**：每次发送新消息后，如何保证缓存一致性？

**方案 A**：删除缓存（简单，推荐）
```java
// 发送消息后，删除消息列表缓存
public void afterSendMessage(String sessionId) {
    String key = "session:messages:" + sessionId;
    redisTemplate.delete(key);
    // 下次查询时会重新从 DB 加载
}
```

**方案 B**：追加到缓存列表（复杂，性能好）
```java
// 发送消息后，直接追加到 Redis 列表
public void appendMessageToCache(String sessionId, Message message) {
    String key = "session:messages:" + sessionId;
    redisTemplate.opsForList().rightPush(key, message);
    // 限制列表长度，最多保留 100 条
    redisTemplate.opsForList().trim(key, -100, -1);
}
```

**推荐**：初期使用方案 A，后续优化时改用方案 B

#### 4. 分布式锁（防止并发问题）

**场景**：用户同时打开两个窗口发送消息，可能导致会话状态不一致

```java
@Service
public class DistributedLockService {
    
    @Autowired
    private RedisTemplate<String, Object> redisTemplate;
    
    /**
     * 尝试获取锁
     */
    public boolean tryLock(String sessionId, long timeoutSeconds) {
        String key = "lock:session:" + sessionId;
        String value = UUID.randomUUID().toString();
        
        Boolean success = redisTemplate.opsForValue()
            .setIfAbsent(key, value, timeoutSeconds, TimeUnit.SECONDS);
        
        return Boolean.TRUE.equals(success);
    }
    
    /**
     * 释放锁
     */
    public void unlock(String sessionId) {
        String key = "lock:session:" + sessionId;
        redisTemplate.delete(key);
    }
}
```

```java
@Configuration
public class RabbitMQConfig {
    
    // 交换机
    public static final String CHAT_EVENT_EXCHANGE = "chat.event.exchange";
    
    // 队列
    public static final String LOG_QUEUE = "chat.event.log";
    public static final String STATS_QUEUE = "chat.event.stats";
    
    // Routing Keys
    public static final String ROUTING_KEY_LOG = "event.log";
    public static final String ROUTING_KEY_STATS = "event.stats";
    
    /**
     * 声明 Topic Exchange
     */
    @Bean
    public TopicExchange chatEventExchange() {
        return new TopicExchange(CHAT_EVENT_EXCHANGE, true, false);
    }
    
    /**
     * 日志队列
     */
    @Bean
    public Queue logQueue() {
        return QueueBuilder.durable(LOG_QUEUE).build();
    }
    
    /**
     * 统计队列
     */
    @Bean
    public Queue statsQueue() {
        return QueueBuilder.durable(STATS_QUEUE).build();
    }
    
    /**
     * 绑定日志队列
     */
    @Bean
    public Binding logBinding(Queue logQueue, TopicExchange chatEventExchange) {
        return BindingBuilder.bind(logQueue)
            .to(chatEventExchange)
            .with(ROUTING_KEY_LOG);
    }
    
    /**
     * 绑定统计队列
     */
    @Bean
    public Binding statsBinding(Queue statsQueue, TopicExchange chatEventExchange) {
        return BindingBuilder.bind(statsQueue)
            .to(chatEventExchange)
            .with(ROUTING_KEY_STATS);
    }
}
```

```java
@Component
```

```java
public class LogConsumer {
    
    @Autowired

    @RabbitListener(queues = RabbitMQConfig.LOG_QUEUE)

            log.error("日志写入 ES 失败", e);
            // 可以选择重试或写入死信队列
        }
    }
}
```

### 🔧 技术要点

#### 1. SessionController API 设计

```java
@RestController
@RequestMapping("/api/v1/sessions")
@Validated
public class SessionController {
    
    @Autowired
    private SessionService sessionService;
    
    /**
     * 创建新会话
     * POST /api/v1/sessions
     */
    @PostMapping
    public Result<SessionVO> createSession(@RequestBody @Valid CreateSessionRequest request) {
        SessionDTO session = sessionService.createSession(request.getUserId(), request.getModelType());
        return Result.success(convertToVO(session));
    }
    
    /**
     * 查询会话列表（分页）
     * GET /api/v1/sessions?page=1&size=20
     */
    @GetMapping
    public Result<Page<SessionVO>> listSessions(
            @RequestParam Long userId,
            @RequestParam(defaultValue = "1") int page,
            @RequestParam(defaultValue = "20") int size) {
        Page<SessionVO> sessions = sessionService.listSessions(userId, page, size);
        return Result.success(sessions);
    }
    
    /**
     * 查询会话详情
     * GET /api/v1/sessions/{sessionId}
     */
    @GetMapping("/{sessionId}")
    public Result<SessionVO> getSessionDetail(@PathVariable String sessionId) {
        SessionVO session = sessionService.getSessionDetail(sessionId);
        return Result.success(session);
    }
    
    /**
     * 删除会话
     * DELETE /api/v1/sessions/{sessionId}
     */
    @DeleteMapping("/{sessionId}")
    public Result<Void> deleteSession(@PathVariable String sessionId) {
        sessionService.deleteSession(sessionId);
        return Result.success();
    }
}
```

#### 2. ChatController API 设计

```java
@RestController
@RequestMapping("/api/v1/chat")
@Validated
public class ChatController {
    
    @Autowired
    private ChatService chatService;
    
    /**
     * 发送消息（同步）
     * POST /api/v1/chat/send
     */
    @PostMapping("/send")
    public Result<ChatResponse> sendMessage(@RequestBody @Valid ChatRequest request) {
        ChatResponse response = chatService.sendMessage(request);
        return Result.success(response);
    }
    
    /**
     * 发送消息（流式响应 - SSE）
     * POST /api/v1/chat/stream
     */
    @PostMapping(value = "/stream", produces = MediaType.TEXT_EVENT_STREAM_VALUE)
    public Flux<ChatResponse> sendMessageStream(@RequestBody @Valid ChatRequest request) {
        return chatService.sendMessageStream(request);
    }
}
```

#### 3. SSE 流式响应实现

```java
@Service
public class ChatServiceImpl implements ChatService {
    
    @Autowired
    private WebClient webClient;
    
    /**
     * 流式响应实现
     */
    @Override
    public Flux<ChatResponse> sendMessageStream(ChatRequest request) {
        String sessionId = request.getSessionId();
        
        // 1. 保存用户消息
        saveUserMessage(sessionId, request.getContent());
        
        // 2. 获取会话上下文
        List<Message> context = getConversationContext(sessionId, 10);
        
        // 3. 调用 AI API（流式）
        return webClient.post()
            .uri("/api/chat/stream")
            .bodyValue(buildPrompt(context, request.getContent()))
            .retrieve()
            .bodyToFlux(String.class)
            .map(chunk -> {
                // 保存 AI 回复片段（可选）
                return new ChatResponse(chunk, false);
            })
            .concatWith(Flux.just(new ChatResponse("[DONE]", true)))
            .doOnComplete(() -> {
                // 流式响应完成后，保存完整的 AI 回复
                saveAssistantMessage(sessionId, fullResponse);
            })
            .onErrorResume(error -> {
                log.error("流式响应出错", error);
                return Flux.just(new ChatResponse("抱歉，服务暂时不可用", true));
            });
    }
}
```

## 📝 常见问题和解决方案

### Q1: Redis 缓存穿透怎么办？
**A**: 对于不存在的 sessionId，缓存一个空对象（TTL 5分钟），避免频繁查询 DB。

### Q2: 消息队列积压如何处理？
**A**: 
1. 增加消费者数量
2. 优化消费者处理逻辑
3. 临时提高队列最大长度
4. 监控告警提前发现

### Q3: 会话上下文太大导致 Prompt 超长？
**A**: 
1. 限制上下文消息数量（最多 10-20 条）
2. 对长消息进行截断或摘要
3. 使用滑动窗口策略

### Q4: 如何保证消息顺序性？
**A**: 
1. 使用单一 Queue，避免并发消费乱序
2. 消息携带序列号，消费者按序处理
3. 对于聊天场景，允许轻微乱序（用户体验影响小）

### Q5: ES 写入性能瓶颈？
**A**: 
1. 使用 Bulk API 批量写入
2. 调整 refresh_interval（默认 1s，可改为 5-10s）
3. 增加 ES 节点数量
4. 优化 Index 映射（减少不必要的字段）
